01 · Context i situació inicial

Un gimnàs 0.0 sense digitalització

El projecte parteix d’un gimnàs tradicional que treballa de manera pràcticament analògica. Els registres de socis, les reserves, els cobraments i la comunicació amb els clients es fan manualment o amb eines poc centralitzades.

Problemes detectats

  • Errors en registres i reserves.
  • Pèrdua o duplicació d’informació.
  • No hi ha una base de dades centralitzada.
  • No existeix una infraestructura de xarxa definida.
  • No hi ha canals digitals potents per als clients.

Valor que aporta el sistema

  • Millora de l’organització interna.
  • Centralització de dades.
  • Web pública accessible.
  • Eines interactives per als usuaris.
  • Base per a futures millores 4.0.

03 · Solucions tècniques

Tecnologies i eines utilitzades

Àrea Tecnologia Justificació
Web pública HTML, CSS, JavaScript Permeten crear una web visual, responsive i amb interacció bàsica.
Calculadores JavaScript Els càlculs s’executen directament al navegador.
Funcionalitat obligatòria PHP Permet processar formularis i connectar amb base de dades.
Base de dades MySQL / MariaDB Adequada per gestionar socis, reserves, activitats i dades LAB.
Infraestructura Ubuntu Server / LAMP / XAMPP Entorn realista per provar serveis web i base de dades.
Gestió del projecte GitHub + GanttLab Permet gestionar issues, milestones, dependències i planificació.

Una decisió important és separar la part estàtica de la web, que pot estar publicada a GitHub Pages o hosting, de la part PHP, que necessita un entorn compatible com XAMPP, LAMP o un hosting amb PHP.

04 · Integració i interacció

Com es connecten les parts del sistema

El sistema està pensat per funcionar per capes. La web pública mostra informació i eines per als clients. El panell intern estaria orientat al personal del gimnàs. La funcionalitat PHP permetria enviar dades cap a la base de dades. La infraestructura de xarxa i servidor dona suport a aquests serveis.

Client / Soci │ ▼ Web pública ── Calculadores JS │ ▼ Formulari PHP │ ▼ Base de dades MySQL/MariaDB │ ├── Reserves ├── Socis ├── Activitats └── Dades LAB Personal / Entrenador │ ▼ Panell intern │ ▼ Gestió i consulta de dades

05 · Planificació del projecte

Fases, milestones i tasques

El projecte s’organitza amb GitHub Issues i Milestones. Cada tasca té un objectiu, una dependència i una data prevista. GanttLab permet visualitzar el calendari i detectar quines tasques bloquegen altres parts del projecte.

Fase 1 — Definició i disseny

Context, abast, actors, casos d’ús, objectius i arquitectura general.

Fase 2 — Disseny tècnic

Infraestructura, xarxa, serveis, base de dades, seguretat i tecnologies.

Fase 3 — Desenvolupament

Web pública, calculadores, funcionalitat PHP, base de dades i panell intern.

Fase 4 — Validació i tancament

Proves, capturas, resultats, conclusions, memòria i presentació final.

08 · English slide

System Architecture

GYM ZERO is designed as a layered system. The public website provides information and interactive tools for users. The internal panel is planned to manage gym data such as members, bookings and LAB records. PHP connects web forms with the database, while the server infrastructure supports internal services, backups and security.

Main components

  • Public website
  • Internal management panel
  • PHP functionality
  • MySQL/MariaDB database
  • Server infrastructure
  • Network segmentation

Technical value

The project connects web development, databases, network design, services and cybersecurity into a single realistic digital transformation proposal.

10 · Conclusions i aprenentatges

Aprenentatges del projecte

Aquest projecte permet entendre que una transformació digital no consisteix només a crear una pàgina web, sinó a dissenyar un sistema complet amb infraestructura, serveis, dades, seguretat i planificació.

Aprenentatges tècnics

  • Disseny de sistemes per capes.
  • Planificació amb GitHub Issues i milestones.
  • Diferència entre web estàtica i PHP.
  • Ús de JavaScript per funcionalitats interactives.
  • Importància de la base de dades i la seguretat.

Dificultats i millores

  • Acotar correctament una idea molt gran.
  • Diferenciar entre simulació i implementació real.
  • Organitzar la documentació tècnica.
  • Completar el mòdul PHP i la base de dades.
  • Afegir evidències i proves finals.
